POST /v1/verify

鉴权验证。其他后端服务调这个接口即可，无需自己存密钥。响应永远 200，按 valid 字段判断。

POST /v1/verify
Content-Type: application/json

{ "key": "sk_live_xxxxxx_xxxxxxxxxxxx" }
POST /v1/verify
Authorization: Bearer sk_live_xxxxxx_xxxxxxxxxxxx
POST /v1/verify
x-api-key: sk_live_xxxxxx_xxxxxxxxxxxx

成功响应

200 OK
{
  "valid": true,
  "keyId": 1,
  "name": "prod-server",
  "meta": { "owner": "team-a" }
}

失败响应

200 OK
{
  "valid": false,
  "code":  "NOT_FOUND | DISABLED | EXPIRED | MISSING_KEY",
  "reason": "key 不存在"
}

业务侧调用示例

curl -X POST http://keymgr:3000/v1/verify \
  -H "Content-Type: application/json" \
  -d '{"key":"'"$MY_KEY"'"}'
// Express 中间件
async function requireKey(req, res, next) {
  const r = await fetch('http://keymgr:3000/v1/verify', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ key: req.headers['x-api-key'] }),
  });
  const { valid, code } = await r.json();
  if (!valid) return res.status(401).json({ error: code });
  next();
}
# Flask
import requests
def require_key(key: str) -> bool:
    r = requests.post(
        'http://keymgr:3000/v1/verify',
        json={'key': key},
        timeout=2,
    )
    return r.json().get('valid') is True
// net/http 中间件
func Verify(next http.Handler) http.Handler {
  return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
    key := r.Header.Get("X-API-Key")
    body, _ := json.Marshal(map[string]string{"key": key})
    resp, _ := http.Post("http://keymgr:3000/v1/verify",
      "application/json", bytes.NewReader(body))
    defer resp.Body.Close()
    var out struct{ Valid bool `json:"valid"` }
    json.NewDecoder(resp.Body).Decode(&out)
    if !out.Valid { http.Error(w, "unauthorized", 401); return }
    next.ServeHTTP(w, r)
  })
}

POST /v1/variables/list

列出所有变量名（不含 value，避免泄露）。

POST /v1/variables/list
Authorization: Bearer sk_live_xxx

→ 200
{
  "ok": true,
  "items": [
    { "name": "DB_URL",   "description": "生产数据库" },
    { "name": "FEATURE_X","description": "灰度开关" }
  ]
}

行为说明

  • 鉴权：必须先通过 Key 验证（status 有效、未停用、未过期），否则返回对应 code。
  • name 格式：2~64 字符，[A-Za-z0-9_.-]，建议大写 + 下划线（如 DB_URL、API_BASE）。
  • value 限制：字符串，≤ 8192 字符（足够存连接串、JSON 序列化配置等）。
  • 限流：1200 次/分钟/IP，/v1/verify 限流独立（600/min）。
  • 不记录 value：日志只记 name，不记 value，避免日志泄露。

POST /api/keys — 导入已有 Key

把外部已经签发的 key 纳入管理。prefix 可省略，自动取前 12 位。

POST /api/keys
{
  "mode":  "import",
  "name":  "openai-imported",
  "plain": "sk-proj-AbCdEfGhIjKlMn12345",
  "prefix":"sk-proj",          // 可选
  "meta":  {"source":"openai"} // 可选
}

→ 200 { "ok":true, "id":1, "prefix":"sk-proj", "imported":true, "message":"已导入 key" }
→ 409 { "ok":false, "code":"KEY_DUPLICATE", "id":3 }   // 哈希已存在
→ 400 { "ok":false, "code":"MISSING_PLAIN",  "field":"plain" }
→ 422 { "ok":false, "code":"PLAIN_TOO_SHORT", "field":"plain" }

POST /api/keys/:id/reroll

重置：生成新明文，旧 key 立即失效，original_plain 保留。

→ 200 { "ok":true, "id":1, "prefix":"sk_live_abc123",
        "key":"sk_live_abc123_新明文", "warning":"旧 key 已失效...",
        "message":"已重新生成 key" }
→ 404 NOT_FOUND

POST /api/keys/:id/toggle

→ 200 { "ok":true, "id":1, "enabled":false }
→ 404 NOT_FOUND

PATCH /api/keys/:id

局部更新（重命名 / 改 meta / 改过期）。任何字段可省略。

PATCH /api/keys/1
{ "name":"new-name", "meta":{...}, "expiresAt":"2027-01-01T00:00:00Z" }

→ 200 { "ok":true, "message":"已更新" }
→ 400 MISSING_NAME / INVALID_META / INVALID_EXPIRES_AT
→ 409 NAME_TAKEN
→ 404 NOT_FOUND

DELETE /api/keys/:id

→ 200 { "ok":true, "message":"已删除" }
→ 404 NOT_FOUND

字段说明

  • name 1~64 字符，全局唯一。
  • prefix 2~24 字符，字符集 [A-Za-z0-9_.\-]。留空使用 sk_live。
  • meta 任意 JSON 对象，存回显时会原样返回给 /v1/verify 调用方。
  • expiresAt ISO 8601 字符串。过期后验证返回 EXPIRED。
  • 明文存储：数据库存 original_plain（创建/导入时）+ current_plain（当前有效），重置时 current 跟着变，original 永久保留。删除时整行清掉。

字段说明

  • name 2~64 字符，[A-Za-z0-9_.-]，全局唯一。业务侧调 /v1/variables/get 时传这个。

状态码语义

状态含义触发场景
200OK成功创建/查询/更新/删除/验证
400Bad Request参数缺失、JSON 解析失败、格式错
401Unauthorized未登录或会话失效
403ForbiddenCSRF token 错
404Not Found资源不存在
409Conflictname 重复 / 导入的 key 哈希已存在
422Unprocessable格式合法但语义错（key 太短、过期时间已过去）
429Too Many Requests限流（默认 600/min/IP，仅 /v1/verify）

错误响应统一格式

{
  "ok": false,
  "error": "name 已存在",
  "code":  "NAME_TAKEN",
  "field": "name"     // 字段级错误时存在，前端用于回填
}

成功响应统一格式

{
  "ok": true,
  "...": "业务字段"
}

code 列表（管理 API）

MISSING_NAME       name 缺失或格式错
INVALID_PREFIX     prefix 格式错
INVALID_META       meta 不是 JSON 对象
INVALID_EXPIRES_AT expiresAt 格式错或已过期
MISSING_PLAIN      导入模式缺 plain
PLAIN_TOO_SHORT    导入的 key 太短（< 8）
IMPORT_FAILED      导入失败
NAME_TAKEN         name 已存在
KEY_DUPLICATE      导入的 key 哈希已存在
NOT_FOUND          资源不存在

（/v1/verify 用另一组：NOT_FOUND/DISABLED/EXPIRED/MISSING_KEY）

认证机制

  • 登录成功后下发 km_sid Cookie（HttpOnly, SameSite=Lax）。
  • 所有 /api/* 需要带该 Cookie，否则 401。
  • 写操作额外需要 X-CSRF-Token 头（值 = GET /api/auth/me 返回的 csrfToken）。
  • /v1/verify 不需要认证，仅限流。

新建 Key

选择创建方式，再填信息。

允许字母/数字/下划线/中划线/点，2~24 字符。留空使用 sk_live。

高级：自定义前缀（可选）

任何前缀都可以，例如 ak_、tk_、xoxb-、sk-proj-。不影响验证。

导入后系统只保存 SHA-256 哈希，明文不会落库。需要再次复制可点列表里的"重置"。
